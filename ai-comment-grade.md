จัดให้ตามคำขอครับ นี่คือตาราง **Trust Level ของซอฟต์แวร์ (Software Reliability)** ที่ประยุกต์เข้ากับบริบทของโค้ดคำนวณเกรดที่คุณให้มา เพื่อให้เห็นภาพว่าการพัฒนาโค้ดจากระดับพื้นฐานไปจนถึงระดับมืออาชีพเป็นอย่างไรครับ

### ตาราง Trust Level ของซอฟต์แวร์ (Software Reliability)

| ระดับความเชื่อถือ | ชื่อระดับ (Status) | คำอธิบาย (Description) | ตัวอย่างจากโค้ดของคุณ |
| :--- | :--- | :--- | :--- |
| **Level 0** | **Unstable (อันตราย)** | โค้ดมี Syntax Error หรือบั๊กที่ทำให้รันไม่ได้เลย หรือค้างทันทีเมื่อเจอข้อมูลผิดปกติ | **โค้ดเริ่มต้น:** มีการย่อหน้าผิด (`IndentationError`) ทำให้โปรแกรมทำงานไม่ได้ |
| **Level 1** | **Functional (ใช้งานได้)** | โค้ดรันได้และทำงานถูกต้องในสภาวะปกติ (Ideal Case) แต่ไม่มีการดักจับข้อผิดพลาด | **โค้ดที่แก้การย่อหน้าแล้ว:** คำนวณเกรดได้ถูกต้อง แต่ถ้าใส่ลิสต์ว่าง `[]` โปรแกรมจะพัง (Crash) |
| **Level 2** | **Reliable (เชื่อถือได้)** | มีการเช็คข้อผิดพลาดพื้นฐาน (Basic Validation) ป้องกันไม่ให้โปรแกรมค้าง | **โค้ดที่มีการเช็คลิสต์ว่าง:** เพิ่ม `if not scores:` เพื่อป้องกันการหารด้วยศูนย์ (ZeroDivisionError) |
| **Level 3** | **Robust (แข็งแกร่ง)** | รองรับข้อมูลที่หลากหลายและผิดปกติได้ (Edge Cases) เช่น ข้อมูลไม่ใช่ตัวเลข หรือคะแนนติดลบ | **โค้ดที่มี Error Handling:** ใช้ `try...except` ดักจับกรณีคนใส่ตัวอักษรแทนตัวเลข และเช็คว่าคะแนนต้องอยู่ช่วง 0-100 |
| **Level 4** | **Professional (ระดับมืออาชีพ)** | โค้ดสะอาด (Clean), ทำงานเร็ว (Optimized), มีคำอธิบาย และพร้อมนำไปใช้งานจริง | **โค้ดที่สมบูรณ์:** ใช้ `sum()`, มี Type Hinting, มี Docstring และเขียนตามมาตรฐาน PEP 8 |

### วิธีขยับ Trust Level ในโค้ดของคุณ (ให้ถึง Level 4)

หากคุณต้องการให้โค้ดของคุณไปถึงระดับสูงสุด นี่คือสิ่งที่ควรทำเพิ่มครับ:

1.  **Validation:** ตรวจสอบข้อมูลก่อนคำนวณเสมอ (เช่น คะแนนต้องไม่ติดลบ และไม่เกิน 100)
2.  **Error Handling:** ใช้ `try...except` เพื่อดักจับความผิดพลาดที่คาดไม่ถึง
3.  **Efficiency:** เลือกใช้วิธีที่สั้นและเร็ว (เช่น ใช้ `sum(scores)` แทนการเขียน Loop บวกเอง)
4.  **Readability:** เขียนชื่อตัวแปรให้ชัดเจน และใส่คำอธิบาย (Type Hinting & Docstring)

### ตัวอย่างโค้ดระดับ "Level 4" (Professional Ready)
ลองดูความแตกต่างเมื่อเราอัปเกรดโค้ดเป็นระดับสูงสุดครับ:

```python
def calculate_grade(scores: list[float]) -> tuple[str, float]:
    """
    คำนวณเกรดเฉลี่ยจากรายการคะแนนที่ได้รับ
    Return: (เกรด, คะแนนเฉลี่ย)
    """
    # 1. Validation & Error Handling (Level 2-3)
    if not scores:
        return "N/A", 0.0
    
    try:
        # 2. Efficiency: ใช้ Built-in function (Level 4)
        average = sum(scores) / len(scores)
        
        # 3. Logic: ตรวจสอบความถูกต้องของช่วงคะแนน
        if not (0 <= average <= 100):
            return "Invalid Score", average

        # 4. Grading Logic
        if average >= 80: grade = "A"
        elif average >= 70: grade = "B"
        elif average >= 60: grade = "C"
        elif average >= 50: grade = "D"
        else: grade = "F"
        
        return grade, round(average, 2)

    except TypeError:
        return "Error: ข้อมูลในลิสต์ต้องเป็นตัวเลขเท่านั้น", 0.0

# การใช้งาน
scores = [85, 92, 78, 88, 95]
print(calculate_grade(scores))
```
